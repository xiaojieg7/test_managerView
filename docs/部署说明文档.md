# 小区物业管理系统 部署说明文档

## 一、项目概述

本系统是基于 Spring Boot + Vue3 的小区物业管理系统，实现了物业管理流程的自动化和信息化。系统包含三种角色：管理员、维修人员和业主，不同角色拥有不同的权限和功能模块。

## 二、技术架构

### 后端
- **框架**: Spring Boot 3.2.5
- **ORM**: MyBatis-Plus 3.5.6
- **数据库**: MySQL 8.0
- **安全**: Spring Security + JWT
- **端口**: 8080

### 前端
- **框架**: Vue3 + Vite + TypeScript
- **状态管理**: Pinia
- **UI组件**: Element Plus
- **HTTP客户端**: Axios
- **端口**: 8081

## 四、部署步骤

### 4.1 环境准备

#### 后端环境
- JDK 17+
- Maven 3.8+
- MySQL 8.0+

#### 前端环境
- Node.js 18+
- npm 9+

### 4.2 数据库配置

#### 4.2.1 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE property_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出
EXIT;
```

#### 4.2.2 执行初始化脚本

```bash
# 进入 sql 目录
cd /workspace/sql

# 执行初始化脚本
mysql -u root -p property_management < init.sql

# 或者在 MySQL 中执行
mysql -u root -p
USE property_management;
SOURCE /workspace/sql/init.sql;
```

#### 4.2.3 修改数据库连接配置

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/property_management?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root          # 修改为你的数据库用户名
    password: root123456    # 修改为你的数据库密码
```

### 4.3 后端部署

#### 4.3.1 IDEA 导入

1. 打开 IntelliJ IDEA
2. File -> Open -> 选择 `/workspace/backend` 目录
3. 选择 Maven 项目，等待依赖下载完成

#### 4.3.2 配置 Maven（可选）

在 `backend/pom.xml` 所在目录执行：

```bash
cd /workspace/backend
mvn clean install
```

#### 4.3.3 启动后端服务

1. 在 IDEA 中打开 `PropertyManagementApplication.java`
2. 点击运行按钮，或使用快捷键 `Shift + F10`
3. 等待服务启动，日志显示 "Started PropertyManagementApplication" 表示成功

#### 4.3.4 验证后端服务

```bash
# 测试健康检查
curl http://localhost:8080/api/auth/login -X POST -H "Content-Type: application/json" -d '{"username":"admin","password":"123456"}'
```

### 4.4 前端部署

#### 4.4.1 安装依赖

```bash
cd /workspace/frontend
npm install
```

#### 4.4.2 启动开发服务器

```bash
npm run dev
```

前端服务将运行在 `http://localhost:8081`

#### 4.4.3 构建生产版本（可选）

```bash
npm run build
```

构建产物将生成在 `frontend/dist` 目录。

### 4.5 访问系统

打开浏览器访问 `http://localhost:8081`

## 七、常见问题

### 7.1 跨域问题

后端已配置 CORS，允许前端开发服务器访问。如仍有问题，检查：
- 后端服务是否正常运行在 8080 端口
- 前端代理配置是否正确

### 7.2 端口占用

如果端口被占用：
```bash
# 查找占用端口的进程
# Windows
netstat -ano | findstr :8080

# Linux/Mac
lsof -i :8080

# 结束进程或修改端口
```

### 7.3 JWT 过期

Token 默认有效期为 24 小时。过期后系统会自动跳转登录页。

### 7.4 权限不生效

1. 检查 Token 是否正确携带在请求头
2. 确认用户角色是否正确配置
3. 检查后端 Security 配置

### 7.5 数据库连接失败

1. 确认 MySQL 服务已启动
2. 检查用户名密码是否正确
3. 确认数据库已创建

### 7.6 前端依赖安装失败

```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

## 八、API 接口说明

### 认证接口
- `POST /api/auth/login` - 用户登录
- `POST /api/auth/logout` - 用户登出

### 管理接口（需管理员权限）
- `GET/POST/PUT/DELETE /api/admin/users` - 用户管理
- `GET/POST/PUT/DELETE /api/admin/owners` - 业主管理
- `GET/POST/PUT/DELETE /api/building` - 楼栋管理
- `GET/POST/PUT/DELETE /api/notice` - 公告管理
- `GET /api/dashboard/stats` - 仪表盘统计

### 业务接口
- `GET/POST /api/repair` - 报修管理
- `PUT /api/repair/{id}/assign` - 分配维修人员
- `PUT /api/repair/{id}/status` - 更新报修状态
- `GET/POST/PUT /api/payment` - 缴费管理
- `POST /api/payment/{id}/pay` - 支付账单

## 九、安全配置

### JWT 配置
修改 `backend/src/main/resources/application.yml` 中的 JWT 配置：

```yaml
jwt:
  secret: your-secret-key    # 修改为更复杂的密钥
  expiration: 86400000       # 24小时（毫秒）
```

### 密码加密
用户密码使用 BCrypt 加密存储，不可逆。
